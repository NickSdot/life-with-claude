# Detail Files

Extended context for entries, stored in `details/{ID}.md`.

## When to Use
- Complex feature wishes needing design context
- Bug reports requiring reproduction steps
- Model behaviour issues
- Anything needing more than a brief description

## IMPORTANT: Use Official Templates

When creating detail files, you MUST follow the structure from:
`templates/github.com_anthropics_claude-code/`

Match entry type to template:
- **Bug** → `bug_report.yml`
- **Wish** → `feature_request.yml`
- **Flaw** → `model_behavior.yml`

## Writing Detail Files

1. Read the matching template
2. Follow its sections and required fields
3. Be thorough but concise
4. Include reproduction steps (numbered, minimal)
5. Sanitise any sensitive data
